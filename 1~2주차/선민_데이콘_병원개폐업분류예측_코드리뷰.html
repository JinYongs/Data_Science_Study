Dacon - 병원 개/폐업 분류 예측¶

  • https://dacon.io/competitions/official/9565/overview/

목적¶

  • 의료기관이 폐업할지 안할지 여부를 예측

데이터 설명¶

  • train.csv - 의료기관이 폐업했는지 여부를 포함하여 **최근 2개년의 재무정보와 병원 기본정보**
  • test.csv - 폐업 여부를 제외하고 train.csv와 동일
  • sample_submission.csv - inst_id와 open과 close를 예측하는 OC 두개의 열로 구성. OC의 값은 open 예측일 경우 1, close 예측일 경우 0.
  • 훈련데이터 301개, 테스트데이터 127개의 비교적 적은 데이터

변수 설명¶

(1) 병원정보 10개¶

  • **inst_id - 각 파일에서의 병원 고유 번호**
  • **OC – 영업/폐업 분류, 2018년 폐업은 2017년 폐업으로 간주함**
  • sido – 병원의 광역 지역 정보
  • sgg – 병원의 시군구 자료
  • openDate – 병원 설립일
  • bedCount - 병원이 갖추고 있는 병상의 수
  • instkind – 병원, 의원(소병원), 요양병원, 한의원, 종합병원 등 병원의 종류

    종합병원 : 입원환자 100명 이상 수용 가능 (general_hospital)
    일반병원 : 입원 환자 30명 이상 100명 미만 수용 가능 (hospital)
    요양병원 : (nursing_hospital) 의원(소병원) : 입원 환자 30명 이하 수용 가능 (clinic)
    한방 병원(한의원) : 침술과 한약으로 치료하는 의료 기관. (traditional_clinic)

  • employee1 – 고용한 총 직원의 수, 2017(회계년도)년 데이터를 의미함
  • employee2 – 고용한 총 직원의 수, 2016(회계년도)년 데이터를 의미함
  • ownerChange – 대표자의 변동

앙상블 (ensemble)¶

  • 앙상블 쓰는 이유 : 여러 개의 단일 모델들의 평균치를 내거나, 투표를 해서 다수결에 의한 결정을 하는 등 더 나은 결과를 도출해 내기 위한 목적기법
  • 배깅방법과 부스팅 방법이 있고, 배깅에는 RandomForest / 부스팅방법에는 AdaBoost, GradientBoost(GBM) , AdaBoost, XGBoost, LightGradientBoost 등이 있음.

(1) Bagging¶

  • 샘플을 여러번 뽑아 각 모델을 학습시켜 결과를 집계
  • 복원추출로 N개의 데이터셋을 만들고 N개의 모델을 만듬
  • 다양한 데이터 셋 사용 하니까 Overfitting 방지
  • Classification과 regression 모두 사용 가능

(1) - 1. 랜덤 포레스트¶

  • 여러 개의 의사 결정 나무를 만들고 그들의 다수결로 결과를 결정하는 방법
  • 성능이 좋고, 함수가 간단해서 하이퍼 파라미터 튜닝 없이 간단히 쓰기 가능
  • 단

(2) Boosting¶

  • 배깅의 변형으로 모델이 잘 예측하지 못하는 부분을 개선하기 위한 모델
  • 이전 모델들이 예측하지 못한 Error data에 가중치를 부여하여, 다음 모델이 더 잘 예측되도록 하는 것.

(3) 배깅 vs 부스팅¶

  • Bagging은 일반적인 모델을 만드는데 집중되어 있는 반면, Boosting은 맞추기 어려운 문제를 맞추는데 초점이 맞춰져 있음.
  • Boosting은 속도가 느리지만 Bagging에 비해 에러가 적고, 데이터를 사용한 걸 재사용하니 오버피팅 될 가능성이 있음.
  • ==> 개별 결정 트리의 낮은 성능이 문제라면 Boosting / 오버피팅이 문제라면 Bagging

threshold = 0.5
def eval_error(preds, dtrain):
    labels = dtrain.get_label()
    preds = (preds > threshold ).astype('float')
    return "accuracy", accuracy_score(labels, preds)
    
# xgboost - reg:linear(linear-regression / binary:logistic classification)

         'max_depth' : 6, # leaf node 2^6 = 64

         'subsample' : 1,
         # 'silent' : 0 -> not be used error
         # eval_metrics : 모델의 평가 함수 조정하는 함수
         }

# xgb_model 학습 
xgb_model = xgb.train(param, 
                      dtrain_prod,
